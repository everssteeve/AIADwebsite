# Préambule

## Pourquoi lire ce préambule ?

Les agents IA ont rendu obsolète la façon dont nous développions des logiciels. Ce préambule explique ce qui a changé et pourquoi les méthodes traditionnelles ne fonctionnent plus.

**Temps de lecture : 5 minutes**

---

## Le constat

### Ce qui a changé

Les agents IA savent désormais :
- Transformer une intention claire en code fonctionnel
- Générer des tests en parallèle du code
- Refactorer et optimiser automatiquement

**Conséquence directe** : la valeur d'un développeur ne réside plus dans sa capacité à écrire du code, mais dans sa capacité à **exprimer une intention précise** et **valider un résultat**.

### Exemple concret

**Avant (approche traditionnelle)**
1. Développeur écrit 500 lignes → 2 jours
2. Code review, corrections → 1 jour
3. Tests écrits après coup → 0.5 jour
4. **Total : 3-4 jours, qualité variable**

**Avec AIAD**
1. Product Engineer rédige une SPEC précise → 30 min
2. Agent génère code + tests → 10 min
3. Validation humaine et ajustements → 1h
4. **Total : 2h, qualité standardisée**

La différence ? **30 minutes de réflexion en amont** remplacent **3 jours de production**.

## Trois principes fondateurs

### 1. L'intention prime sur l'exécution

| Haute valeur | Faible valeur |
|--------------|---------------|
| Clarifier le problème à résoudre | Écrire du code |
| Rédiger des spécifications précises | Maîtriser une syntaxe |
| Valider que le résultat répond au besoin | Produire vite |

**En pratique** : une SPEC de qualité évite des jours de corrections. Un agent sans contexte produit du code générique inutilisable.

### 2. Observer plutôt que planifier

Les plans détaillés deviennent obsolètes avant leur exécution. La seule approche viable :

```
Essayer → Observer → Adapter → Répéter
```

**Exemple** :
- ❌ Planifier 3 mois avec des estimations détaillées
- ✅ Définir une direction, livrer en 3 jours, mesurer, ajuster

### 3. L'écosystème dépasse l'individu

Un Product Engineer avec un écosystème d'agents bien configuré surpasse un développeur expert travaillant seul.

| Investissement | Retour |
|----------------|--------|
| Former un expert | Amélioration linéaire (1 personne) |
| Optimiser l'écosystème d'agents | Amélioration multiplicative (toute l'équipe) |

**En pratique** : configurer l'AGENT-GUIDE est l'investissement au meilleur rendement.

## Erreurs fréquentes

### "L'IA va juste nous faire coder plus vite"

**Le problème** : on génère plus de code médiocre, plus vite. La dette technique explose.

**La réalité** : l'IA change *ce qu'on fait*, pas seulement *la vitesse*.

### "Pas besoin de documenter, l'IA comprend"

**Le problème** : code générique, incohérent avec l'existant, corrections manuelles permanentes.

**La réalité** : plus l'agent a de contexte (AGENT-GUIDE, SPEC), plus son output est pertinent.

### "On garde nos Sprints, on ajoute l'IA"

**Le problème** : friction entre la vélocité des agents et la rigidité des cérémonies.

**La réalité** : les boucles itératives fluides remplacent les cycles rigides.
