# Argumentaire Décideur : Pourquoi adopter AIAD ?

**Cible** : CTO, Tech Lead, Engineering Manager, Directeur Technique
**Objectif** : Convaincre de tester le Framework AIAD et son Mode Opératoire

---

## Le constat : vos équipes utilisent déjà l'IA, mais sans méthode

Vos développeurs ont adopté Claude Code, Cursor ou GitHub Copilot. C'est une évidence. Mais posez-vous ces questions :

- **Combien de code généré par l'IA est retravaillé ou jeté ?**
- **Qui maintient la cohérence architecturale quand l'IA génère du code ?**
- **Comment mesurez-vous le ROI réel de ces outils ?**
- **Vos pratiques Scrum sont-elles encore adaptées à cette vélocité ?**

Si vous n'avez pas de réponse claire, vous êtes dans la majorité. Et vous perdez probablement du temps et de l'argent.

---

## AIAD : la méthodologie qui structure l'ère des agents IA

**AIAD** (AI-Agent Iterative Development) est un framework de développement produit conçu nativement pour l'orchestration d'agents IA.

### Ce que AIAD n'est pas

| Idée reçue | Réalité |
|------------|---------|
| "Encore un framework agile" | AIAD rompt avec Scrum/Kanban. Pas de sprints figés, pas de cérémonies rituelles |
| "De la théorie abstraite" | Le Mode Opératoire fournit des étapes concrètes, des prompts prêts à l'emploi, des templates |
| "Réservé aux experts IA" | Conçu pour être adoptable en 1 semaine par une équipe, maîtrisé en 3 mois |

### Ce qu'AIAD apporte

| Bénéfice | Impact concret |
|----------|----------------|
| **Structure l'utilisation des agents IA** | Fin du "vibe coding" improductif. Chaque interaction IA est intentionnelle |
| **Redéfinit les rôles** | Le Product Engineer orchestre l'IA au lieu de coder manuellement. Focus sur la valeur, pas sur la frappe |
| **Maintient la qualité** | Artefacts clairs (PRD, ARCHITECTURE, SPECS) qui servent de contexte aux agents (clairement définis dans un artefact AGENT) |
| **Mesure les gains** | Métriques de productivité, qualité, efficacité IA, outcomes business |
| **Accélère le time-to-market** | Boucles itératives continues au lieu de sprints rigides |

## Les 5 questions qu'un décideur se pose

### 1. "Quel ROI puis-je attendre ?"

Les équipes utilisant AIAD reportent :

- **Réduction du temps de développement** : moins de code à écrire manuellement
- **Moins de retravail** : les agents IA reçoivent un contexte clair (AGENT-GUIDE, SPECS)
- **Qualité maintenue** : processus de validation structuré (DoOD, DoOuD)

> Le ROI se mesure dès le premier projet pilote. AIAD inclut des métriques concrètes pour le démontrer.

### 2. "Comment s'intègre-t-il avec nos pratiques actuelles ?"

AIAD est **modulaire** :

- Vous pouvez adopter uniquement les artefacts (PRD, SPECS, AGENT-GUIDE)
- Vous pouvez garder vos outils actuels (Jira, GitHub, etc.)
- La transition peut être progressive : commencez par un projet pilote

AIAD complète vos pratiques agiles existantes. Il ne les remplace que si vous le souhaitez.

### 3. "Combien de temps pour être opérationnel ?"

| Phase | Durée | Résultat |
|-------|-------|----------|
| **Compréhension** | 30 minutes | Lecture du framework compris |
| **Démarrage** | 1 semaine | Équipe opérationnelle sur un projet pilote |
| **Maîtrise** | 3 mois | Expertise acquise, pratiques intégrées |

Le Mode Opératoire guide chaque étape : de l'installation de Claude Code jusqu'au déploiement en production.

### 4. "Mes équipes vont-elles résister au changement ?"

AIAD valorise les développeurs, il ne les remplace pas :

- Le rôle de **Product Engineer** est une évolution de carrière, pas une menace
- Les développeurs passent du "comment coder" au "quoi faire coder"
- Le temps libéré est réinvesti dans la réflexion, la revue, l'architecture

> Les équipes qui adoptent AIAD reportent une satisfaction en hausse : moins de tâches répétitives, plus d'impact.

### 5. "Par où commencer ?"

**Option 1 : Exploration rapide (2 heures)**
1. Lisez le Framework AIAD (parties Vision et Artefacts)
2. Évaluez la pertinence pour votre contexte
3. Identifiez un projet pilote candidat

**Option 2 : Pilote encadré (1 semaine)**
1. Suivez la Phase d'Initialisation du Mode Opératoire
2. Configurez un projet avec les artefacts AIAD
3. Mesurez les premiers résultats

**Option 3 : Adoption accompagnée**
1. Formation de l'équipe sur le Framework
2. Implémentation guidée sur un projet réel
3. Rétrospective et ajustement

## Les signaux qui indiquent qu'AIAD est pour vous

- Vos développeurs utilisent des agents IA mais sans méthode commune
- Vous constatez des incohérences dans le code généré par l'IA
- Vos sprints Scrum semblent inadaptés à la vélocité des agents IA
- Vous cherchez à mesurer objectivement les gains de productivité
- Vous voulez structurer l'adoption des agents IA avant que le chaos s'installe

---

## Les signaux qui indiquent qu'AIAD n'est pas (encore) pour vous

AIAD sera moins pertinent si :

- Votre équipe n'utilise pas encore d'agents IA de codage
- Vous êtes en contexte très contraint (compliance stricte interdisant les LLMs)
- Votre équipe est inférieure à 2-3 personnes et fonctionne déjà bien en mode artisanal

> AIAD s'adresse aux équipes prêtes à industrialiser leur usage des agents IA.
